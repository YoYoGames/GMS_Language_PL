gracz_lokalny

Gdy użyjesz rollback_define_player() do zdefiniowania gracza object, system automatycznie tworzy instancję tego object dla każdego gracza. Każda instancja tego object otrzymuje zmienną player_local, która mówi, czy instancja gracza jest lokalna, czy nie.

Jeśli nie używasz rollback_define_player(), ta zmienna nie zostanie utworzona dla żadnej strony objects.

Możesz użyć tej zmiennej do wskazania, który gracz jest lokalny. Nie jest zalecane używanie tej zmiennej w sposób wpływający na logikę gry, która powinna być taka sama dla graczy lokalnych i zdalnych.

Składnia:

player_id;

Zwroty:

Boolean

Przykład:

// Draw event
if (player_local)
{
    draw_sprite(spr_arrow, 0, x, y - 60);
}

Powyższy kod jest uruchamiany w zdarzeniu Draw i sprawdza, czy instancja gracza jest lokalna. W takim przypadku rysuje strzałkę sprite nad instancją, informując gracza, którą instancją steruje.
