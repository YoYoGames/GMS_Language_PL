Preferencje dotyczące żywienia w czasie rzeczywistym Preferencje Runtime Feeds służą do wyświetlania i zmiany aktualnie zainstalowanego programu Runtime oraz aktualnie używanych narzędzi runtime. GameMaker Studio 2 może mieć w danej chwili zainstalowanych kilka różnych narzędzi runtime i w tych preferencjach można określić, które z nich mają być używane. Najlepiej byłoby, gdybyś używał najbardziej aktualnego runtime, ale czasami może się zdarzyć, że wystąpi jakiś problem, albo że będziesz musiał użyć starszego narzędzia dla celów porównawczych itp. Dlatego możesz tutaj zmienić narzędzia runtime, których GameMaker Studio 2 używa do tworzenia gry. Pierwsza sekcja tych preferencji dotyczy miejsca, z którego pobierane są narzędzia runtime i wymaga wprowadzenia adresu URL do kanału RSS. Kanał ten będzie następnie skanowany w poszukiwaniu aktualizacji narzędzi runtime, które można następnie pobrać i zainstalować. W tej sekcji domyślnie wyświetlany jest główny adres URL kanału runtime, ale można kliknąć przycisk Dodaj , aby utworzyć nowy kanał Runtime, podając nazwę i adres URL kanału RSS, a następnie kliknąć przycisk Zastosuj, aby umożliwić dostęp do niego portalowi GameMaker Studio 2. Aby nowy kanał został wyświetlony, trzeba będzie zamknąć i ponownie otworzyć okno Preferencje. Po utworzeniu kanału Runtime będzie on wymieniony w tej sekcji jako podkategoria. Zawsze powinieneś mieć przynajmniej jeden kanał na liście (domyślnie jest to kanał główny ). Niezależnie od używanego kanału runtime, wszystkie one będą miały te same funkcje, opisane poniżej na przykładzie kanału Master: Preferencje Runtime Feed służą do wybierania, instalowania i odinstalowywania różnych narzędzi runtime. U góry znajduje się informacja o aktualnie używanych narzędziach runtime, dacie utworzenia i miejscu przechowywania na dysku twardym. Pod tymi informacjami znajduje się lista wszystkich dostępnych narzędzi runtimes, z której można wybrać dowolne narzędzie, aby stało się ono "działającym" runtime używanym do kompilacji projektów. Aby wybrać inną stronę runtime do użycia, wystarczy znaleźć ją na liście i kliknąć dwukrotnie . Jeśli strona runtime nie została jeszcze pobrana i zainstalowana, zostanie wyświetlony monit o jej pobranie, a po zainstalowaniu zostaniesz poproszony o zamknięcie i ponowne uruchomienie strony IDE, aby nowa strona runtime mogła zostać zainicjowana. Lista ta zawiera wszystkie dostępne runtimes, numer wersji, datę wydania oraz - jeśli są zainstalowane - adres path, pod którym zostały zainstalowane. Po prawej stronie wyświetlane są także różne ikony stanu, które przedstawiają się następująco: Oznacza bieżącą stronę runtime, która została wybrana do użycia podczas tworzenia projektów. Kliknięcie tego przycisku spowoduje rozpoczęcie pobierania i instalowania wybranej strony runtime (w prawym górnym rogu IDE zostanie wyświetlony pasek stanu). Kliknięcie tego przycisku spowoduje usunięcie wybranej strony runtime z dysku twardego. Listę runtime można w każdej chwili zaktualizować z podanego kanału RSS, klikając przycisk odświeżania , a dysk twardy można wyczyścić ze starych plików runtimes, klikając przycisk "Wyczyść teraz Runtimes ".
