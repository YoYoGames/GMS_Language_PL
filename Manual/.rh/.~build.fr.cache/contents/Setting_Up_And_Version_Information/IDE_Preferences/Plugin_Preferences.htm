Preferencje wtyczek Preferencje wtyczek służą do kontrolowania wszelkich dodatkowych wtyczek, które zostały dodane do GameMaker Studio 2 IDE. Domyślnie jedna z sekcji jest poświęcona wtyczce Kontrola źródła, która służy do integracji projektów GameMaker Studio 2 z Git. Kontrola źródeł (Git) W tej sekcji omówiono preferencje wymagane do współpracy wtyczki Git SCM z witryną GameMaker Studio 2. Ścieżka do pliku wykonywalnego Git: Jest to lokalizacja pliku wykonywalnego Git, którego GameMaker używa do wykonywania wszystkich poleceń Source Control. Jak pokazano na powyższym obrazku, będzie on znajdował się w folderze cmd w instalacji Git. Poniżej można skonfigurować narzędzie Scalanie i narzędzie Różnica dla strony Source Control z tymi ustawieniami(opcjonalnie): Merge (Lokalizacja narzędzia): W tym miejscu można określić pełny plik path do narzędzia do scalania Git. Merge (Opcje narzędzia): W tym polu można dodać dowolne dodatkowe polecenia, które będą uruchamiane przy każdym użyciu narzędzia Scalanie. Różnica (położenie narzędzia): W tym miejscu określany jest pełny plik path do narzędzia Git diff. Różnica (opcje narzędzia): W tym polu można dodać dowolne dodatkowe polecenia, które mają być uruchamiane przy każdym użyciu narzędzia diff.
