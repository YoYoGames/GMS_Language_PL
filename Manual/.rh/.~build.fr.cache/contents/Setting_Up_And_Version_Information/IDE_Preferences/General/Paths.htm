Preferencje ścieżki Preferencje ścieżki kontrolują różne pliki paths wymagane przez elementy w witrynie GameMaker Studio 2 IDE, a także przez zewnętrzne edytory, których użytkownik potrzebuje. W IDE dostępne są następujące opcje: Katalog tymczasowy: Miejsce, w którym zapisywane są wszystkie foldery tymczasowe. Domyślnie na stronie Windows jest to \Users\\AppData\Local\Temp\GameMakerStudio2 A w witrynie macOS jest to: /var/folders//GameMakerStudio2 Katalog IDE cache: Miejsce zapisania pliku IDE cache. Domyślnie na stronie Windows jest to \Users\\AppData\Roaming\GameMakerStudio2\Cache A na stronie macOS jest to: /Users//.config/GameMakerStudio2/Cache Katalog Asset cache: Lokalizacja, w której zapisywane są zasoby asset cache dla każdego projektu. Domyślnie na stronie Windows jest to \Users\\AppData\Roaming\GameMakerStudio2\Cache Natomiast na stronie macOS jest to: /Users//.config/GameMakerStudio2/Cache Lokalizacja "Moje projekty": Lokalizacja, w której GameMaker Studio 2 będzie początkowo tworzyć nowe projekty. Domyślnie na stronie Windows jest to \Users\\Documents\GameMakerStudio2 Natomiast w witrynie macOS jest to /Users//GameMakerStudio2 Automatycznie usuwaj katalog tymczasowy przy zamykaniu: Zaznaczenie tej opcji spowoduje, że GameMaker Studio 2 będzie automatycznie usuwać folder Temp, który jest tworzony w każdym projekcie do kompilacji itp. To ustawienie jest domyślnie włączone, a odznaczenie go spowoduje switch go wyłączyć (co oznacza, że później trzeba będzie ręcznie usunąć wszystkie pliki tymczasowe). Automatycznie usuwaj asset cache przy zamykaniu: Jeśli ta opcja jest zaznaczona, folder asset kompilatora cache zostanie usunięty po zamknięciu GameMaker Studio 2. Domyślnie opcja ta jest wyłączona, a jej włączenie spowoduje, że przy każdym załadowaniu i uruchomieniu przy każdym załadowaniu i uruchomieniu projektu konieczne będzie ponowne utworzenie folderu cache (co może zająć dużo czasu w zależności od rozmiaru gry). Automatycznie usuwaj IDE cache przy zamykaniu: Jeśli ta opcja jest zaznaczona, folder IDE kompilatora cache zostanie usunięty po zamknięciu GameMaker Studio 2. Domyślnie opcja ta jest wyłączona, a jej włączenie będzie oznaczało, że przy każdym uruchomieniu programu GameMaker Studio 2 trzeba będzie odbudowywać folder cache. Usuń folder tymczasowy: Kliknięcie tego przycisku spowoduje usunięcie folderu tymczasowego dla projektu. Usuń Asset Cache: Kliknięcie tego przycisku spowoduje usunięcie kompilatora asset cache dla danego projektu. Usuń pamięć podręczną IDE: Kliknięcie tego przycisku spowoduje usunięcie pamięci podręcznej IDE. Istnieją następujące opcje ustawiania adresu paths dla zewnętrznych edytorów: Ścieżka do zewnętrznego edytora/przeglądarki dla plików SWF: Jeśli pracujesz z plikami SWF w formacie sprites, możesz ustawić tę opcję na path preferowanej przeglądarki/edytora, a po kliknięciu przycisku Edytuj obraz w Edytorze Sprite zostanie zostanie otwarty dany program. Domyślną wartością jest brak adresu path.
