Pliki Biblioteka działań Akcje plikowe dotyczą dwóch różnych typów plików - Buffer oraz Ini - a także zawierają pewne ogólne akcje plikowe służące do zmiany nazwy lub kopiowania istniejących plików. Pliki buforowe tworzy się, zapisując dane z utworzonej wcześniej strony buffer. Dane te można następnie w dowolnym momencie w przyszłości ponownie załadować do strony buffer. Więcej informacji na temat strony buffers można znaleźć tutaj: Akcje buforowe. Pliki Ini to małe, lekkie pliki, które są kompatybilne z większością platform. Są idealne do przechowywania niewielkich informacji, takich jak preferencje interfejsu, lokalne wyniki, dane poziomów itp. Pliki Ini nie muszą można odczytać z nieistniejącego pliku Ini i po prostu otrzymać wartość zwrotną default (określoną przez użytkownika), jednak zalecamy utworzenie co najmniej "bazowego" pliku ini, który będzie można otwierać i modyfikować przed użyciem akcji. Taki bazowy plik ini można utworzyć, wywołując po prostu akcję Open Ini File, a następnie Close Ini File, ponieważ zamknięcie plik zostanie zapisany na dysku, lub można go umieścić w folderze Included Files (Dołączone pliki ) w przeglądarce Asset. Jeśli używasz pliku dołączonego do Asset Browser jako bazowego Ini, powinieneś również przeczytać rozdział podręcznika dotyczący działania systemu plików. UWAGA: W przypadku gier z lokalizacjami bardzo ważne jest, aby akcentowane litery mogły być odczytywane z plików zewnętrznych. Oznacza to, że należy najpierw utworzyć plik ini w formacie UTF8, a następnie dodać go do programu GameMaker jako plik dołączony aby był on eksportowany podczas uruchamiania gry i używany zamiast domyślnego pliku ini w formacie ANSI, który jest tworzony przez funkcje ini, jeśli wcześniej nie podano żadnego pliku. W ten sposób można poprawnie odczytywać i zapisywać w nim tekst z zachowaniem wszystkich akcentów i liter nieromańskich. zachowane są wszystkie akcenty i litery nieromanowe.
