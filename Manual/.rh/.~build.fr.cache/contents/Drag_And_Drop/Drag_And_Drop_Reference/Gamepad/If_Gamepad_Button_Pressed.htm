Jeśli naciśnięto przycisk Gamepad Za pomocą tej akcji można sprawdzać, czy na gamepadzie jest aktualnie wciśnięty jakiś przycisk. Akcja zwróci true w każdym kroku, gdy przycisk gamepada jest wciśnięty (i przytrzymany), lub false w przeciwnym razie, choć jeśli klikniesz modyfikator not można sprawdzić, czy przycisk nie jest wciśnięty, tzn. akcja zwróci true, jeśli nie jest wciśnięty żaden przycisk, i false, jeśli jest wciśnięty. Jeśli chcesz sprawdzić, czy naciśnięto tylko jeden przycisk, to użyj akcji if gamepad button down. Jeśli sprawdzany przycisk jest przyciskiem analogowym, naciśnięcie nie zostanie wykryte, dopóki wartość przycisku nie przekroczy progu ustawionego za pomocą akcji Ustaw próg przycisku gamepada. Należy pamiętać, że aby dodać akcje do bloku "jeśli", należy je upuścić z boku akcji, jak pokazano na poniższym rysunku: Te działania będą teraz uruchamiane, jeśli "jeśli" zakończy się wynikiem true, podczas gdy wszelkie działania porzucone w innym miejscu zostaną wykonane po bloku "jeśli". Składnia działania: Argumenty: Argument Opis Gamepad Indeks gamepada. Button Nazwa przycisku, który ma być sprawdzany. Not Negacja sprawdzenia (prawda staje się fałszem i odwrotnie) Przykład: Powyższy kod bloku akcji co krok sprawdza stan przycisków na gamepadzie 0 i jeśli przycisk jest przytrzymywany, sprawdza, czy nie nastąpiło pierwsze naciśnięcie przycisku. Jeśli sprawdzane jest true początkowe naciśnięcie przycisku, odtwarzany jest dźwięk, a następnie, gdy przycisk jest przytrzymany, prędkość instancji jest ustawiana na 10. Jeśli przycisk nie jest nie jest przytrzymywany, po zwolnieniu przycisku sprawdzana jest prędkość, aby ponownie ustawić ją na 0.
