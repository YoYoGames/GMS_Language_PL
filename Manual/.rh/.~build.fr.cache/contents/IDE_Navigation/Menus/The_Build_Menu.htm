Menu budowania W menu budowania można wybrać opcję budowania projektu do testów, na stronie debugging lub jako ostateczny plik wykonywalny. Dostępne opcje to: Uruchom - uruchamia bieżący projekt na wybranej platformie docelowej w celu przetestowania. Informacje o różnych opcjach dostępnych podczas kompilacji można znaleźć w rozdziale Kompilacja. Należy pamiętać, że to polecenie jest tożsame z naciśnięciem przycisku Uruchom w witrynie IDE. Debugowanie - uruchom bieżący projekt do testowania za pomocą strony debugger, aby uzyskać szczegółowe informacje o wydajności i usunąć ewentualne błędy lub problemy. Więcej informacji na ten temat można znaleźć na stronie debugging, w rozdziale Debugowanie. Należy pamiętać, że to polecenie jest tożsame z naciśnięciem przycisku Debug w witrynie IDE. Czyszczenie - GameMaker przechowuje pliki w pamięci podręcznej, aby przyspieszyć czas kompilacji podczas testowania. Czasami jednak "nieświeży" cache może prowadzić do dziwnych i nieoczekiwanych błędów w grze. W takim przypadku pierwszą rzeczą, którą powinieneś zrobić, jest wyczyszczenie cache i ponowne przetestowanie. Dobrym pomysłem jest również wyczyszczenie cache przed utworzeniem ostatecznego pliku wykonywalnego dla platformy docelowej. Pamiętaj, że to polecenie jest tożsame z naciśnięciem przycisku Clean Cache na stronie IDE. Create Executable (Utwórz plik wykonywalny ) - to polecenie skompiluje bieżący projekt i utworzy pakiet (lub pakiety) wykonywalny dla platformy docelowej. Najpierw zostanie otwarty eksplorator plików, aby można było wybrać miejsce, w którym projekt ma zostać zapisany, a następnie gra zostanie skompilowana. Więcej informacji na temat konfigurowania celów kompilacji znajduje się w rozdziale Kompilacja. To polecenie jest tożsame z naciśnięciem przycisku Create Executable (Utwórz plik wykonywalny ) na stronie IDE.
