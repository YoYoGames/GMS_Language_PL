Podstawowa struktura kodu Typowy blok kodu składa się z zestawu instrukcji, zwanychinstrukcjami, które są następnie interpretowane przez program GameMaker i używane do wykonania czegoś w grze. To "coś" może być tak proste, jak dodanie 2 i 2, aby otrzymać 4, lub tak złożone, jak spowodowanie, że wróg ucieknie, gdy jego zdrowie spadnie poniżej pewnej wartości. Struktura programu może być bardzo różna, w zależności od funkcji, z których korzysta, ale w uproszczeniu wygląda to tak: ; ; ... Wyrażenia powinny być oddzielone znakiem ';', aby zapobiec błędom w deklaracjach zmiennych i zachować czystość kodu. Mogą one składać się z deklaracji zmiennych, wyrażeńi wywołań określonych funkcji. Można również "grupować" wyrażenia w bloki, używając nawiasów klamrowych {}, aby działały razem, tak jak w poniższym przykładzie warunkowym: if () {     ;     ;     ... } UWAGA Strona GameMaker Language przyjmie również adresy begin oraz end zamiast nawiasów klamrowych {}, choć nie jest to najczęściej stosowany sposób: if () begin     ;     ;     ... end Oto bardziej wizualne przedstawienie wyglądu bloku kodu, tym razem utworzonego jako skrypt w edytorze skryptów GameMakera: Istnieje wiele różnych typów twierdzeń, wyrażeń, wyrażeń warunkowych i funkcji, z których wszystkie zostały szczegółowo omówione w kolejnych częściach podręcznika. UWAGA Jeśli nie masz doświadczenia w programowaniu, przed kontynuacją warto zapoznać się z podręcznikiem Szybki start.
