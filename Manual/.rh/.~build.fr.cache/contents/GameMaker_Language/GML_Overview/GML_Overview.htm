Przegląd języka GameMaker Ta część instrukcji zawiera wszystkie informacje potrzebne do zrozumienia i używania języka skryptowego GMLprogramu GameMaker Studio 2. Funkcje runtime w tym języku mogą być używane do tworzenia gier i są dodawane do objects z Edytora obiektów, zwykle po wybraniu opcji tworzenia nowego projektu GameMaker Language Project, choć można go także używać razem z DnD™ (Drag and Drop). Poniżej widać typowy obraz strony object z edytorem kodu otwartym na zdarzeniu: Każde zdarzenie ma swoją własną zakładkę w edytorze i w każdej chwili można dodawać, edytować lub usuwać z nich kod (więcej informacji o zdarzeniach można znaleźć w dziale Zd arzenia obiektu). Sam kod musi mieć podstawową strukturę i może zawierać indeksy zasobów, zmienne, funkcje, wyrażenia, słowa kluczowe itp. Jeśli jesteś początkujący w programowaniu lub tworzeniu switch z DnD, zalecane jest, abyś zaczął od strony poświęconej podstawowej strukturzekodu, a następnie zapoznał się z wszystkimi pozostałymi stronami w tej sekcji, testując kod z każdej z nich w samymGameMaker Studio 2.
