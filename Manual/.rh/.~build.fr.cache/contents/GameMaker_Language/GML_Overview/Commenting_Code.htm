Kod komentowania Podczas pracy we współpracy z innymi osobami, przy dużych projektach, a nawet dla własnych potrzeb i debugging, pozostawianie komentarzy w kodzie zdarzeń i na stronie scripts jest bardzo ważne. W GameMaker Studio 2 masz do dyspozycji różne mechanizmy ułatwiające zostawianie notatek i komentarzy dotyczących fragmentów kodu, a nawet całych bloków kodu, które są widoczne na liście zdarzeń object, którą właśnie edytujesz. W ten sposób można zostawiać notatki dla siebie i kolegów, wyjaśniać szczególnie skomplikowane fragmenty kodu lub po prostu przypominać, co coś robi lub co ma robić w przyszłości. Aby zostawić prosty, jednowierszowy komentarz, należy przede wszystkim użyć adresu // przed tekstem. Na przykład: //initialize variables sr = 10; hp = 100; Można również zostawiać wielowierszowe komentarze, aby wyrazić uznanie, pominąć cały fragment kodu na stronie debugging, a nawet zostawić uwagi o tym, jak chcesz zmienić kod w przyszłości. Do tego celu można użyć /* ... */ na przykład tak: /* Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values) IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables. */ frame_num = 30; frame_steps = 5; frame_width = 256; frame_height = 256; frame_quality = gifcam_quality_high; Oprócz komentarzy można również używać specjalnych znaczników #region do oznaczania fragmentów kodu do składania, a każdy z tych fragmentów może zawierać komentarz. Działa to w ten sposób, że dodajesz znacznik #region - wraz z tekstem komentarza - na górze sekcji kodu, a następnie dodajesz odpowiadający mu znacznik #endregion na dole sekcji i możesz "złożyć" kod, tzn. ukryć go, aby zmniejszyć liczbę linii kodu widocznych w edytorze w danym momencie: Rzeczywisty kod wygląda mniej więcej tak: #region This is an example region width = sprite_get_width(sprite); height = sprite_get_height(sprite); xoff = sprite_get_xoffset(sprite); yoff = sprite_get_yoffset(sprite); #endregion Further comment here if required Na koniec warto wspomnieć, że podczas pracy ze zdarzeniami w Edytorze obiektów można użyć specjalnego komentarza "description" na początku zdarzenia, aby powiedzieć, jaka jest ogólna zawartość zdarzenia, co będzie widoczne w oknie zdarzeń Edytora Object. Komentarz ten powinien znaleźć się w pierwszej linii edytora kodu i jest sformatowany w następujący sposób: /// @description Zostanie to teraz wyświetlone w oknie zdarzeń, np: Uwaga: w Preferencjach edytora obiektów można zdefiniować domyślny komentarz, który będzie tworzony dla wszystkich zdarzeń. Wreszcie, jeśli piszesz funkcje skryptu i chcesz mieć kontrolę nad tym, jak są one traktowane przez IDE (np. mieć autouzupełnianie lub pokazywać parametry script ), powinieneś pisać komentarze, używając notacji JSDoc, jak wyjaśniono tutaj.
