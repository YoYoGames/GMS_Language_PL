System operacyjny i kompilator Podczas tworzenia gier wieloplatformowych bardzo ważne jest uzyskanie szczegółowych informacji o systemie operacyjnym używanym przez urządzenie, na którym uruchomiona jest gra. Takie informacje, jak język, wersja czy stan sieci, mogą być wykorzystane do dostosowania gry do systemu, na którym działa, i zapewnienia użytkownikom jak najlepszych wrażeń. Dlatego GameMaker Studio 2 zawiera wiele specjalnych funkcji, które można wykorzystać do uzyskania niezbędnych informacji. Poniższe funkcje i zmienne można wykorzystać do uzyskania pewnych informacji o systemie operacyjnym lub przeglądarce, w której uruchomiona jest gra: os_browser os_device os_type os_version os_is_paused os_is_network_connected os_get_config os_get_language os_get_region os_get_info os_powersave_enable os_lock_orientation os_check_permission os_request_permission Niektóre funkcje są prekompilowane podczas uruchamiania gry (tzn. są używane do definiowania sposobu, w jaki zostanie skompilowana ostateczna wersja gry), a niektóre są również używane po skompilowaniu gry w celu uzyskania szczegółowych informacji o środowisku runtime. Tutaj można znaleźć pełną listę tych funkcji: GM_build_date GM_version GM_runtime_version gml_release_mode gml_pragma parameter_count parameter_string environment_get_variable W witrynie Windows rozdzielczość harmonogramu wątków Windows jest domyślnie ustawiona na 1 ms podczas uruchamiania gry. Poniższe funkcje umożliwiają pobieranie i zmianę rozdzielczości harmonogramu wątków Windows w czasie uruchamiania gry: scheduler_resolution_get scheduler_resolution_set Wreszcie, podczas pracy z rozszerzeniami może być potrzebna możliwość definiowania i wywoływania zewnętrznych funkcji na stronie runtime, dlatego należy użyć następującego rozwiązania: UWAGA: Są to starsze funkcje i wszelkie funkcje extension należy definiować w samym edytorze Extension.
