ds_list_destroy Funkcja ta usuwa podaną strukturę danych listy z pamięci, zwalniając zasoby, z których korzystała i usuwając wszystkie wartości, które zawierała. Funkcja ta powinna być zawsze używana po zakończeniu korzystania z ds_list, aby zapobiec wyciekom pamięci, które mogą spowolnić i zawiesić grę. Zauważ, że zniszczenie listy spowoduje usunięcie odniesienia do wszystkich struktur danych przechowywanych w niej, co spowoduje wyciek pamięci, więc aby temu zapobiec, musisz ręcznie przejrzeć listę i zniszczyć wszystkie elementy struktur danych przed jej zniszczeniem. Jedynym przypadkiem, w którym nie jest to wymagane, jest oznaczenie elementów na liście jako innej listy DS lub mapy DS. W takim przypadku elementy te zostaną zniszczone, a ich pamięć automatycznie wyczyszczona. WAŻNE! Gdy tworzysz strukturę danych, wartość indeksu identyfikująca ją jest liczbą całkowitą zaczynającą się od 0. Oznacza to, że struktury danych różnych typów mogą mieć tę samą wartość indeksu, więc jeśli masz wątpliwości, powinieneś użyć funkcji ds_exists() przed uzyskaniem do nich dostępu. Należy również pamiętać, że indeksy są używane ponownie, więc wartość indeksu zniszczonej struktury danych może zostać użyta przez nowo utworzoną strukturę. Dlatego zalecamy, aby po zniszczeniu zawsze ustawiać zmienną, która przechowywała indeks DS, na -1. Składnia: ds_list_destroy(id); ArgumentTyp Opis id The id of the data structure to remove. Zwroty: Przykład: if (lives == 0) {     ds_list_destroy(AI_list);     AI_list = -1;     room_goto(rm_Menu); } Powyższy kod sprawdzi wartość wbudowanej zmiennej globalnej "lives" i jeśli będzie ona równa 0, zniszczy listę DS indeksowaną w zmiennej "AI_list", a następnie zmieni adres rooms.
