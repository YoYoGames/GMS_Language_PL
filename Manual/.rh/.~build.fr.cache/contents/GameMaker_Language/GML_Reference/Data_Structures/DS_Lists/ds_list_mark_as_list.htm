ds_list_mark_as_list Ta funkcja "zaznacza" (lub "oflagowuje") daną pozycję w uprzednio utworzonej liście DS jako zawierającą inną listę DS. Funkcja ta jest wymagana podczas kodowania JSON strings (który można utworzyć za pomocą json_encode()), ale może być również przydatna podczas zagnieżdżania struktur danych, ponieważ pozycje oznaczone w ten sposób będą automatycznie usuwane (niszczone), gdy zostanie zniszczona nadrzędna lista DS. Oznacza to, że nie trzeba ręcznie przeglądać zawartości listy i niszczyć zaznaczonych struktur danych pojedynczo przed zniszczeniem listy "rodzica". Jeśli jednak pozycja listy zostanie usunięta pojedynczo, lista DS, którą zawierała, nie zostanie odśmiecona i przed usunięciem elementu z pozycji listy nadrzędnej należy wywołać odpowiednią funkcję niszczącą listę DS. Należy także pamiętać, że jeśli wywołasz funkcję ds_list_clear() na liście, wszystkie pozycje oznaczone jako listy zostaną również zniszczone podczas kasowania listy. Warto również zauważyć, że jeśli pozycja, która została zaznaczona, zmieni się w obrębie listy, "znak" przesunie się wraz z nią, więc jeśli zaznaczono pozycję 3 na liście (na przykład), a następnie wstawiono pod nią 2 kolejne elementy, tak że przesunęła się ona na pozycję 5, nadal będzie zaznaczona jako lista. Składnia: ds_list_mark_as_list(id, pos); ArgumentTyp Opis id The id of the list to mark. pos The position within the list to mark. Zwroty: DS List ID or -1 (if it fails) Przykład: var j_list = ds_list_create(); var sub_list = ds_list_create(); ds_list_add(sub_list, health); ds_list_add(sub_list, lives); ds_list_add(sub_list, score); ds_list_add(j_list, sub_list); ds_list_mark_as_list(j_list, 0); Powyższy kod tworzy dwie listy DS, a następnie wypełnia jedną z nich różnymi wartościami ze zmiennych globalnych. Lista ta jest następnie dodawana do drugiej listy, a pozycja jest "oznaczana" jako taka, aby można ją było później poprawnie zakodować.
