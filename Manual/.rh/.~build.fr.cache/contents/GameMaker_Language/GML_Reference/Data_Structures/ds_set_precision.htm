ds_set_precision Podczas porównywania wartości, na przykład podczas wyszukiwania na mapie lub sortowania listy, GameMaker Studio 2 musi zdecydować, kiedy dwie wartości są równe. Dla strings i wartości całkowitych jest to oczywiste, ale w przypadku liczb rzeczywistych, z powodu błędów zaokrąglania liczb zmiennoprzecinkowych, pozornie równe liczby mogą łatwo stać się nierówne. Na przykład może się zdarzyć, że (5 / 3) * 3 nie będzie równe 5! Aby tego uniknąć, we wszystkich funkcjach liczb rzeczywistych używana jest wartość precyzji i gdy różnica między dwiema liczbami jest mniejsza od tej precyzji, są one uważane za równe. Domyślnie dla wszystkich funkcji struktur danych używana jest precyzja 0.0000001, chyba że zostanie zmieniona przez tę funkcję. UWAGA: Ta precyzja jest używana we wszystkich strukturach danych, ale nie w innych porównaniach w GML! Składnia: ds_set_precision(prec); Argument Typ Opis prec Real The precision value (default 0.0000001) Zwroty: N/A Przykład: ds_set_precision(0.0001); Powyższy kod zmieni domyślne ustawienie precyzji dla wszystkich funkcji struktur danych.
