Kolejki DS. Strona DS queue jest nieco podobna do stosu DS, ale działa na zasadzie FIFO (first-in first-out). Wartość, która jako pierwsza zostanie wprowadzona na stronę queue, będzie również jako pierwsza z niej usunięta. Można o tym myśleć jak o queue w sklepie, gdzie osoba, która jako pierwsza wejdzie na queue, płaci jako pierwsza, a następnie wychodzi ze sklepu. Queues są zwykle używane do przechowywania działań lub zachowań (na przykład w postaci scripts), które muszą być jeszcze wykonane przez instancję, ale istnieje wiele innych zastosowań. UWAGA: Podobnie jak wszystkie zasoby dynamiczne, struktury danych zajmują pamięć, dlatego powinny być zawsze niszczone, gdy nie są już potrzebne, aby zapobiec wyciekom pamięci, które spowolnią, a w końcu doprowadzą do awarii gry. UWAGA: jeśli chcesz sprawdzić, czy dana struktura danych istnieje, możesz użyć funkcji ds_exists() funkcji.
