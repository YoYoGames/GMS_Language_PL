ds_stack_read Za pomocą tej funkcji można odtworzyć zapisany stos DS (taki, który został wcześniej zapisany jako string przy użyciu ds_stack_write()). Musisz najpierw utworzyć nowy stos DS, do którego wczytasz string, a jeśli stos DS już istnieje i są w nim zapisane informacje, to zostaną one wyczyszczone przed odczytem. Funkcja ta ma kluczowe znaczenie przy tworzeniu mechanizmów zapisywania i wczytywania danych w grze. Zwróć uwagę, że jeśli określony plik DS string został napisany w programie GameMaker: Studio 1.2.x runtime (lub starszym), należy podać opcjonalny argument "legacy", ustawiając go na true, ponieważ format string zmienił się po tej wersji. Składnia: ds_stack_read(id, str [, legacy]); Argument Typ Opis id Identyfikator stosu DS The id of the data structure to read into. str String The string to read from. legacy Boolean OPTIONAL Można podać adres true lub false albo całkowicie go pominąć. Zwroty: N/A Przykład: stack = ds_stack_create(); ini_open("save.ini"); var str = ini_read_string("Stacks", "0", ""); if (str != "") {     ds_stack_read(stack, str); } ini_close(); Powyższy kod tworzy stos i przechowuje jego indeks w zmiennej "stos". Następnie otwiera plik ini i odczytuje z niego plik string, sprawdzając najpierw, czy plik string nie został zwrócony jako pusty. Ten plik string jest następnie wczytywany do nowo utworzonego stosu DS . Powrót: ds_stack_create Następny: ds_stack_write © Copyright YoYo Games Ltd. 2022 Wszystkie prawa zastrzeżone.
