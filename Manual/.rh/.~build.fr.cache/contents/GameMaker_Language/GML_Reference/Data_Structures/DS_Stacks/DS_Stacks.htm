Stosy DS Stos DS to tak zwana struktura danych typu LIFO (last-in-first-out), podobna do kolejki DS. Na stos można wrzucać wartości, a następnie je usuwać, odrzucając je ze stosu, przy czym wartość, która została wrzucona na stos jako ostatnia, jest pierwszą, która zostanie z niego wyrzucona (pomyśl o stosie monet, gdzie każdą dodaną monetę trzeba najpierw wyjąć, zanim będzie można dostać się do pozostałych monet pod spodem). Stosy są często używane do obsługi przerwań, funkcji rekurencyjnych, a nawet do konstruowania podstawowej sztucznej inteligencji w grach. UWAGA: Podobnie jak wszystkie zasoby dynamiczne, struktury danych zajmują pamięć, dlatego powinny być zawsze niszczone, gdy nie są już potrzebne, aby zapobiec wyciekom pamięci, które spowolnią, a w końcu doprowadzą do awarii gry. UWAGA: Jeśli chcesz sprawdzić, czy dana struktura danych istnieje, możesz użyć funkcji ds_exists() funkcji.
