show_debug_overlay Za pomocą tej funkcji można na stronie switch włączać i wyłączać standardową nakładkę debugowania podczas testowania gry. Nakładka debugowania pokazuje graficzny pasek użycia CPU/GPU w samym oknie gry wraz z bieżącą wartością fps, liczbą texture swaps i liczbą vertex batches (zauważ, że texture swaps i vertex batches nigdy nie będą równe zero i zwykle będą pokazywać wartości 2 lub 3, ponieważ nawet przy pustym room i braku objects GameMaker wciąż musi rysować i wsadować rzeczy). Pasek ten jest podzielony na sekcje, a każda sekcja ma długość 1/60 sekundy. Jak widać na powyższym obrazku, pasek składa się z różnych kolorów: Zielony - Przetwarzanie wejścia/wyjścia (np. klawiatura, mysz, gamepad, sieć itd.) Czerwony - szybkość aktualizacji zdarzenia kroku Żółty - czas wymagany do wykonania losowania Pomarańczowy - Czas aktualizacji debugowania, który jest zwykle widoczny tylko wtedy, gdy używany jest moduł debugowania Biały - czas pozostały do końca renderowania przez GPU, czyli czas oczekiwania na zakończenie renderowania klatki przez GPU przed rozpoczęciem renderowania następnej. Cyjan - czas renderowania tekstu Szary - czas wymagany do oczyszczenia ekranu na każdym etapie losowania Niebieski - czas potrzebny na uruchomienie Garbage Collectora Ciemnoczerwony - Czas przepłukiwania GPU, czyli czas potrzebny GPU na wyczyszczenie obrazów z pamięci. Dzięki tej funkcji można dodać nakładkę niezależnie od tego, czy gra działa w trybie debugowania, czy nie. W ten sposób można zobaczyć, jak wydajnie działa gra, i uzyskać wizualne wskazówki dotyczące wykorzystania dostępnych zasobów, bez kosztów ogólnych związanych z działaniem strony debugger. UWAGA: Ta funkcja nie będzie działać w przypadku celu HTML5 ze względu na inny sposób jego działania niż w przypadku pozostałych celów. Składnia: show_debug_overlay(enable); ArgumentTyp Opis enable switch on (true) lub wyłączone (false) the debug overlay. Zwroty: Przykład: if global.debug {     show_debug_overlay(true); } else {     show_debug_overlay(false); } Powyższy kod włączy lub wyłączy debugowanie w zależności od wartości zmiennej globalnej.
