debug_event Ta funkcja generuje niestandardowe zdarzenie debugowania, które będzie wyświetlane w widoku wykresu na stronie debugger, gdy gra jest uruchomiona w trybie debugowania. Jeśli chcesz, aby komunikaty były wyświetlane, gdy gra nie działa w trybie debugowania, użyj funkcji show_debug_message(). Funkcja pobiera również dwa zarezerwowane adresy strings, aby ułatwić wykonywanie debugging przy użyciu zewnętrznych aplikacji, takich jak Visual Studio. Te strings to: "OutputDebugOn" - Umożliwia wywołanie adresu OutputDebugString dla celu w systemie Windows, co oznacza, że wszystkie dane wyjściowe debugowania - wszystko, co widać w oknie Output - mogą być przeglądane przez Visual Studio lub aplikacje innych firm. "OutputDebugOff" - wyłącza opisane powyżej zachowanie. "BreakOnError" - Ta opcja jest przeznaczona tylko dla kompilacji Windows YYC i oznacza, że projekty nie będą wyświetlać zwykłego ekranu błędu kodu, jeśli runtime wykryje błąd, ale zamiast tego po prostu kontynuują i ulegają awarii. Dzięki temu można zobaczyć ślad stacktrace w Visual Studio, jeśli debugging. "ResourceCounts" - Lista wszystkich aktualnie aktywnych zasobów, takich jak struktury danych, źródła czasu, powierzchnie itp. "DumpMemory" - Podaje informacje o bieżącym wykorzystaniu pamięci. Składnia: debug_event(string) Argument Typ Opis string String The custom debug event string to use. Zwroty: N/A Przykład: if !surface_exists(global.EffectsSurface) {     debug_event("Recreating Effects Surface");     global.EffectsSurface = surface_create(room_width, room_height); } Powyższy kod sprawdza, czy powierzchnia istnieje, a jeśli nie, w widoku wykresu na stronie debugger wywoływane jest zdarzenie debugowania (aby było ono widoczne, gra musi być uruchomiona w trybie debugowania) i powierzchnia jest tworzona na nowo.
