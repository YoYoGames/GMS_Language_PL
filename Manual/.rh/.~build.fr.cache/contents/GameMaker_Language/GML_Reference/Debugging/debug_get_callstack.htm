debug_get_callstack Ta funkcja generuje tablicę strings jako "stos wywołań", w którym najpierw jest wyświetlana bieżąca strona script, a następnie wszystkie inne strony scripts, które zostały uruchomione w celu wykonania bieżącej strony script. Dokładny format string zależy od wybranej platformy docelowej, ale najczęściej będzie zawierał nazwę skryptu/zdarzenia, następnie dwukropek : i numer wiersza, podobnie jak tutaj: "gml_Script_script2:1" "gml_Script_script1:32" "gml_Script_script0:64" "gml_Object_object0_Create_0:1" Funkcja pozwala na przekazanie opcjonalnego argumentu, którym jest maksymalna głębokość zwracanego stosu wywołań. Wartość ta jest liczbą stron scripts, które są zawarte w zwróconej tablicy, poczynając od bieżącej script. Jeśli ten argument nie zostanie podany, zwrócony zostanie pełny stos wywołań. Zwróć uwagę, że zwrócona tablica zawsze będzie zawierała 0 na ostatniej pozycji, po wypisaniu stosu wywołań scripts. Składnia: debug_get_callstack([maxdepth]) Argument Typ Opis [maxdepth] Real OPTIONAL Maksymalna głębokość zwracanego stosu wywołań Zwroty: Array Przykład: if debug_mode {     if keyboard_check(vk_escape)     {         var _a = debug_get_callstack(4);         for (var i = 0; i < array_length_id(_a); ++i;)         {             show_debug_message(_a[i]);         }     } } Powyższy kod sprawdza, czy tryb debugowania jest włączony, a jeśli tak, to czy nie jest przytrzymywany jakiś klawisz. W takim przypadku wypisuje na konsolę bieżący stos wywołań o maksymalnej głębokości 4.
