gpu_set_blendmode_ext_sepalpha Gdy w witrynie GameMaker Studio 2 chcemy narysować piksel, mamy do dyspozycji kolor źródłowy (kolor piksela, który chcemy narysować) i kolor docelowy (kolor, który już znajduje się w pikselu, do którego chcemy narysować), dlatego podczas określania nowego koloru piksela kolory źródłowy i docelowy są obliczane zgodnie z wybranym trybem mieszania. Każdy składnik kolorów jest przechowywany jako wartość zmiennoprzecinkowa z przedziału od 0 do 1, a nowy kolor jest obliczany przez pomnożenie każdego składnika koloru źródłowego przez pewien współczynnik oraz przez pomnożenie każdego składnika koloru docelowego przez inny współczynnik, a następnie zsumowanie wyników składnik po składniku. Ta funkcja umożliwia ustawienie różnych części składowych, które powinny być faktoryzowane razem w celu utworzenia niestandardowego trybu mieszania, wybierając różne części różnych współczynników trybu mieszania dla składowych RGB i alfa. Argumenty "src" i "dest" użyją pierwszych trzech wartości trybu mieszania (RGB), a argumenty "alphasrc" i "alphadest" wezmą ostatnią część współczynnika trybu mieszania. Na przykład, jeśli źródłem RGB jest bm_zero, to użyty zostanie współczynnik (0, 0, 0). Jeśli następnie ustawisz źródło alfa jako bm_one, użyta zostanie wartość (1), co da ostateczny źródłowy tryb mieszania (0, 0, 0, 1). Poniższy wykres przedstawia wartości RGBA dla każdego trybu mieszania, gdzie (Rs, Gs, Bs, As) są wartościami źródłowymi, a (Rd, Gd, Bd, Ad) wartościami docelowymi. W przypadku użycia tej funkcji jedenaście dostępnych stałych może być źródłowych lub docelowych (lub obu). Stała Współczynnik mieszania (czerwony, zielony, niebieski, alfa) bm_zero (0, 0, 0, 0) bm_one (1, 1, 1, 1) bm_src_colour (Rs, Gs, Bs, As) bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As) bm_src_alpha (As, As, As, As) bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As, 1-As) bm_dest_alpha (Ad, Ad, Ad, Ad, Ad) bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad, 1-Ad) bm_dest_colour (Rd, Gd, Bd, Ad) bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad) bm_src_alpha_sat (f, f, f, 1) gdzie f = min(As, 1-Ad) WAŻNE! HTML5 bez włączonej obsługi WebGL nie będzie w stanie poprawnie wyświetlać następujących trybów: bm_src_colour bm_inv_src_colour bm_dest_colour bm_inv_dest_colour bm_src_alpha_sat Zauważ, że do tej funkcji można podać cztery pojedyncze argumenty lub tablicę argumentów (zwracaną przez funkcję gpu_get_blendmode_ext_sepalpha() na przykład). W przypadku podania tablicy powinna ona zawierać następujące cztery elementy: [0] = źródłowy tryb mieszania pomniejszony o składową alfa (domyślnie bm_src_alpha (As, As, As)) [1] = docelowy tryb mieszania pomniejszony o składową alfa (domyślnie bm_inv_src_alpha (1-As, 1-As, 1-As)) [2] = Składowa alfa źródła (domyślnie bm_src_alpha (As)) [3] = Docelowy składnik alfa (domyślnie bm_inv_src_alpha (1-As)) Aby w pełni wykorzystać możliwości trybów mieszania oraz zrozumieć ich działanie i wpływ na ostateczny obraz rysowany na ekranie, zalecamy zapoznanie się z poniższym przewodnikiem: Przewodnik korzystania z Blendmodes Składnia: gpu_set_blendmode(src, dest, alphasrc, alphadest); Argument Typ Opis src Stała współczynnika trybu mieszania Source blend mode factor for RGB (see constants above). dest Stała współczynnika trybu mieszania Destination blend mode factor for RGB (see constants above) alphasrc Stała współczynnika trybu mieszania Source blend mode factor Alpha (see constants above). alphadest Stała współczynnika trybu mieszania Destination blend mode factor Alpha (see constants above) Zwroty: N/A Przykład: var bm = gpu_get_blendmode_ext_sepalpha(); bm[0] = bm_src_alpha; bm[1] = bm_inv_src_alpha; gpu_set_blendmode_ext_sepalpha(bm); Powyższy kod tworzy lokalną tablicę i pobiera bieżący rozszerzony tryb mieszania z oddzielnymi wartościami alfa. Tablica ta jest następnie manipulowana i zwracana w celu ustawienia trybu mieszania.
