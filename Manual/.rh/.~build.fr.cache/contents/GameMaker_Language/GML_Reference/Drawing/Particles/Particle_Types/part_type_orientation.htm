orientacja_typu_części Ta funkcja służy do określania orientacji cząstki sprite podczas jej tworzenia, a także może być używana do zwiększania lub zmniejszania orientacji cząstki w czasie jej życia. Minimalne i maksymalne wartości orientacji wynoszą domyślnie 0, ale można je zmienić, aby losowo zmieniać orientację zgodnie ze standardowymi kierunkami GameMaker Studio 2: 0 stopni w prawo, 90 stopni w górę, 180 stopni w lewo i 270 stopni w dół. Jeśli ustawisz je na tę samą wartość, wszystkie cząstki będą tworzone z tą samą orientacją. Można także ustawić wartość przyrostu, która będzie dodawać (jeśli jest liczbą dodatnią) lub odejmować (jeśli jest liczbą ujemną) pewną liczbę stopni do orientacji w czasie jej trwania. Wartość ta może wynosić co najmniej (+/-) 0,01. Można również ustawić współczynnik "wiggle", tak jak w przypadku innych funkcji cząstek. Jest to wartość, która będzie losowo dodawana lub odejmowana do orientacji w każdym kroku życia cząstki. Oczywiście większe wartości są bardziej wyraźne niż mniejsze, a wartość ta może być nawet ujemna, przy czym maksymalny zakres wynosi od -20 do 20. Wreszcie, można wybrać orientację względną lub nie, co oznacza, że gdy cząstka ma kierunek (i prędkość), cząstka sprite będzie zorientowana wokół tego wektora. Należy pamiętać, że jeśli prędkość cząstki zostanie zmniejszona i osiągnie 0, to brak prędkości spowoduje ustawienie kierunku na domyślną wartość 0°, a zatem orientacja względna spowoduje, że cząstka sprite "przeskoczy" pod innym kątem. Składnia: part_type_orientation( ind, ang_min, ang_max, ang_incr, ang_wiggle, ang_relative ); Argument Typ Opis ind Typ cząstki ID The index of the particle type to change. ang_min Real The minimum starting angle of the particles. ang_max Real The maximum starting angle of the particles. ang_incr Real The increase (or decrease if set negative) of the angle in degrees per step. ang_wiggle Real The amount the angle wiggles clockwise/counterclockwise per step. ang_relative Real Whether to set its angle relative to the direction of the particle's motion (true) or not (false). Zwroty: N/A Przykład: part_type_shape(particle2, pt_shape_spark); part_type_size(particle2, 0.10, 0.50, 0.01, 0); part_type_scale(particle2, 0.30, 0.30); part_type_colour1(particle2, 8454143); part_type_alpha1(particle2, 0.50); part_type_speed(particle2, 4, 4, -0.07, 1); part_type_direction(particle2, 0, 359, 0, 20); part_type_orientation(particle2, 0, 359, 0, 20, 1); part_type_blend(particle2, 1); part_type_life(particle2, 1, 5); Powyższy kod ustawi różne wartości cząstek, w tym orientację, która będzie miała losową wartość z przedziału od 0º do 359º. Ponadto w każdym kroku będzie do niej dodawana losowa wartość z przedziału od 0 do 20, a orientacja będzie odnosiła się do kierunku ruchu.
