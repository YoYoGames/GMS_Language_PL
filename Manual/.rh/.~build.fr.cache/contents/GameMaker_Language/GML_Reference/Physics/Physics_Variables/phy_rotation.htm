phy_rotation Tej zmiennej można użyć do uzyskania (lub ustawienia) kąta nachylenia uchwytu instancji w stopniach, podobnie jak w przypadku ustawiania lub uzyskiwania zmiennej image_angle. Zauważ jednak, że w świecie fizyki obroty są obliczane w odwrotny sposób niż w normalnym świecie gry GameMaker, co oznacza, że funkcje wektorowe takie jak point_direction()powinny mieć zmodyfikowane wartości zwracane (po prostu zamiana wartości dodatniej na ujemną powinna rozwiązać ten problem). Składnia: phy_rotation; Zwroty: (single precision floating point value, or undefined if the instance is not physics enabled) Przykład: if (phy_speed_x > 0) || (phy_speed_y > 0) {     phy_rotation += sqrt(sqr(phy_speed_x) + sqr(phy_speed_y)) /10; } Powyższy kod sprawdza prędkość liniową i jeśli któryś z wektorów nie jest równy 0, oblicza rzeczywistą prędkość i wykorzystuje ją do ustawienia obrotów.
