widok_bieżący Ta zmienna tylko do odczytu jest ważna tylko w przypadku Zdarzenie losowania i zwraca bieżący port widoku, który jest renderowany. Wartość zwracana będzie się zmieniać podczas zdarzenia draw, gdy masz różne widoki, ponieważ zdarzenie draw jest wywoływane raz dla każdego portu widoku po kolei. Tak więc, gdy (na przykład) używasz portów widoków[0] i portów widoków[1] w swojej grze room, zdarzenie draw dla WSZYSTKICH instancji zostanie uruchomione dwukrotnie, raz dla każdego portu, a dzięki tej zmiennej możesz sprawdzić, jaki widok jest aktualnie rysowany. Ogólnie rzecz biorąc, jest to używane do renderowania tylko określonych szczegółów w pojedynczym porcie, gdy wiele portów widoku jest widocznych na stronie room w tym samym czasie. Zobacz przykładowy kod poniżej. Składnia: view_current; Zwroty: Real (view port index from 0 to 7) Przykład: if view_current == 0 {     var xx = camera_get_view_x(view_camera[0]);     var yy = camera_get_view_y(view_camera[0]);     draw_text(xx + 32, yy + 32, "Player 1"); } else {     var xx = camera_get_view_x(view_camera[1]);     var yy = camera_get_view_y(view_camera[1]);     draw_text(xx + 32, yy + 32, "Player 2"); } Powyższy kod sprawdza, który widok jest aktualnie rysowany, a następnie na podstawie wartości zwracanej rysuje inny tekst dla każdego widoku.
