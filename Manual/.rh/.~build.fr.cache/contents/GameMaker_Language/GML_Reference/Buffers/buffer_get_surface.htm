bufor_get_surface Za pomocą tej funkcji można zapisywać informacje z powierzchni do określonego adresu buffer. Strona buffer musi być utworzona wcześniej i powinna być stroną buffer o wyrównaniu 1-bajtowym, wystarczająco dużą, aby pomieścić dane powierzchni, którą zamierzasz zapisać (jeśli nie masz pewności, użyj bufora grow lub zobacz przykład na dole tej strony). Informacje dotyczące powierzchni będą zawsze zapisywane na początku strony buffer (niezależnie od pozycji szukania), ale można wybrać wartość przesunięcia (w bajtach), aby przesunąć pozycję zapisu z początku strony buffer. Zwróć uwagę, że funkcja zapisuje każdy piksel powierzchni do buffer przy użyciu formatowania RGBA w celu Windows, ale w innych celach może być ono inne w zależności od systemu operacyjnego lub nawet urządzenia. UWAGA: Ta funkcja uległa zmianie od aktualizacji GameMaker Studio w wersji 2.3.1. Poprzednie wersje używały stałej buffer_surface_copy, która jest obecnie wycofywana. Składnia: buffer_get_surface(buffer, surface, offset); ArgumentTyp Opis buffer The index of the buffer to use. surface The index of the surface to use. offset The data offset value (in bytes). Zwroty: Przykład: buffer = buffer_create(surf_width * surf_height * 4, buffer_fixed, 0); buffer_get_surface(buffer, surface, 0); Ten kod utworzy stronę buffer o stałym rozmiarze, która może pomieścić 4 bajty na piksel (dla składowych R, G, B i A) w oparciu o dokładny rozmiar powierzchni, a następnie skopiuje do niej dane powierzchni.
