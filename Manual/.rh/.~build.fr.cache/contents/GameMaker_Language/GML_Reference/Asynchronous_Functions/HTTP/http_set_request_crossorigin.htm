http_set_request_crossorigin Za pomocą tej funkcji można ustawić typ pochodzenia krzyżowego, który będzie używany podczas ładowania obrazów z pliku. Funkcja ta jest przeznaczona wyłącznie dla platformy HTML5 i jest przydatna tylko podczas ładowania sprites z pliku za pomocą funkcji sprite_add(). Zwróć uwagę, że po ustawieniu opcji "use-credentials" nie trzeba już podawać bezwzględnego adresu path dla ładowanej strony sprite, lecz względny adres path (jak pokazano w poniższym przykładzie). Domyślnie typ pochodzenia krzyżowego jest ustawiony na "anonymous". Składnia: http_set_request_crossorigin(origin_type); ArgumentTyp Opis origin_type The cross origin type to use (a string) Zwroty: Przykład: if string_lower(http_get_request_crossorigin()) != "use-credentials" {     http_set_request_crossorigin("use-credentials"); } sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0); Powyższy kod najpierw sprawdzi aktualnie ustawiony typ pochodzenia krzyża i jeśli nie jest to "use-credentials", to zostanie ustawiony na "use-credentials", a następnie z pliku zostanie dodany adres sprite.
