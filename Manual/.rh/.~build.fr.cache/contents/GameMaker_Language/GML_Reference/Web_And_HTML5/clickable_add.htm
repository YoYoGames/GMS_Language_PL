clickable_add Za pomocą tej funkcji można utworzyć niestandardową, klikalną ikonę, która działa jako "przycisk" poza oknem gry, ale w oknie przeglądarki. Funkcja zwróci wartość ID, którą należy zapisać w zmiennej i wykorzystać we wszystkich kolejnych funkcjach przycisku, które odwołują się do nowej klikalnej ikony. Funkcja ta jest bardzo przydatna, ponieważ zapobiega tworzeniu przez przeglądarkę wyskakujących okienek po kliknięciu na przycisk, gdyż wszystko to odbywa się za pośrednictwem strony GameMaker Studio 2. Najpierw należy ustawić pozycję, która jest względna w stosunku do (0,0) okna przeglądarki (lewy górny róg), a następnie podać na stronie texture wpis dotyczący strony sprite, na której ma się znajdować ikona przycisku (zob: sprite_get_tpe()). Następnie można podać adres URL do otwarcia lub adres script (z przeglądarki Asset w witrynie GameMaker Studio 2) do otwarcia lub uruchomienia. Parametr "target", który podajesz, jest taki sam, jak standardowa wartość "name" w JavaScript, gdy używasz metody open() (pamiętaj, że wszystkie oprócz "_self" mogą spowodować zablokowanie przeglądarki lub zapytanie użytkownika, czy chce na to pozwolić), a argument "params" jest taki sam, jak parametr "spec" w JavaScript do kontrolowania, jakie właściwości powinno mieć nowe okno/zakładka (nie wszystkie przeglądarki mogą obsługiwać wszystkie funkcje). Jeśli ikona jest używana do uruchamiania wewnętrznego script, te dwa ostatnie argumenty można pozostawić puste strings "". Ważne cele to: Cel Opis _blank Otwiera połączony dokument w nowym oknie lub karcie. _sama Otwiera połączony dokument w tej samej ramce, w której został kliknięty (jest to ustawienie domyślne). _rodzic Otwiera połączony dokument w ramce nadrzędnej. _top Otwiera połączony dokument w pełnym oknie. Dostępne parametry to: Parametry Opis 'height=[px]' Wysokość okna, przy czym minimalna wartość to 100. 'width=[px]' Szerokość okna, przy czym minimalna wartość to 100. 'left=[px]' Lewe położenie okna. 'top=[px]' Górna pozycja okna (tylko IE). 'location=[boolean]' Określenie, czy pole adresu ma być wyświetlane, czy nie (domyślnie 1). 'menubar=[boolean]' Określenie, czy pasek menu ma być wyświetlany (domyślnie 1). 'resizable=[boolean]' Określenie, czy okno ma możliwość zmiany rozmiaru (domyślnie 1). 'scrollbars=[boolean]' Określenie, czy mają być wyświetlane paski przewijania (domyślnie 1). 'status=[boolean]' Określenie, czy ma być dodawany pasek stanu (domyślnie 1). 'titlebar=[boolean]' Określenie, czy ma być wyświetlany pasek tytułowy. Jest to ignorowane, chyba że wywoływana aplikacja jest aplikacją HTML lub zaufanym oknem dialogowym (domyślnie 1); 'toolbar=[boolean]' Określenie, czy ma być wyświetlany pasek narzędzi przeglądarki (domyślnie tak). Warto również zauważyć, że jeśli podany argument URL jest nazwą funkcji skryptu (tzn. nie zaczyna się od "http://"), a adres script nazywa się gmcallback_*, można uzyskać bezpośrednie zasilanie zdarzenia html5. Nazwy funkcji skryptu w programie GameMaker Studio 2 nie są ukryte, jeśli zaczynają się od gmcallback_. Składnia: clickable_add(x, y, tpe, url, target, params); Argument Typ Opis x Real The x position within the window. y Real The y position within the window. tpe Strona tekstury Wprowadzenie The texture page entry for the sprite to be used. url String The URL (website address) to link to, or the script to be run. target String This is the target area to open the URL in (see description). params String The various parameters for the icon (see description). Zwroty: Clickable ID Przykład: home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0"); Powyższy kod tworzy klikalną ikonę DOM na pozycji (32, 32) strony, na której uruchomiona jest gra canvas. Ikona używa adresu sprite, do którego odwołuje się strona texture jako "spr_MS_Home". Po kliknięciu ikona otworzy nowe okno o podanym adresie URL i zdefiniowanych dla niego właściwościach.
