katalog_robotniczy Working_directory może w rzeczywistości zwracać dwie różne wartości, w zależności od tego, do czego jest używana. Jeśli zapisujesz plik na dysku, working_directory wskazuje na obszar pamięci lokalnej zarezerwowany dla Twojej gry na urządzeniu docelowym (w Windows jest to "%LOCALAPPDATA%/gamedir/", gdzie "gamedir" to katalog z nazwą Twojej gry). Jednak w przypadku odczytu z dysku adres working_directory może oznaczać lokalny magazyn (game_save_id) lub obszar, w którym przechowywane są dołączone pliki. UWAGA Zmienna working_directory zwróci adres path wraz z końcowym odwrotnym ukośnikiem. Na przykład, jeśli do gry dołączony jest domyślny plik *.txt i wczytasz go do pamięci za pomocą working_directory, plik ten zostanie pobrany z obszaru, w którym przechowywane są pliki Included Files. Jeśli następnie zdecydujesz się zapisać te informacje do pliku tekstowego za pomocą working_directory, plik zostanie zapisany w pamięci lokalnej platformy docelowej, skąd zostanie odczytany następnym razem, chyba że poinstruujesz GameMakera, by usunął plik, w którym to przypadku working_directory ponownie wskaże obszar, w którym przechowywane są pliki Included Files. Warto zauważyć, że na ogół nie trzeba z tego korzystać i wystarczy podać samą nazwę pliku (wraz z dodatkowymi informacjami path ), co jest wystarczające. Na przykład to: file_text_open_read(working_directory + "my_file.txt"); lepiej byłoby zrobić to w ten sposób: file_text_open_read("my_file.txt"); OSTRZEŻENIE Ta funkcja może nie działać zgodnie z oczekiwaniami użytkownika, ponieważ GameMaker jest sandboxed! Więcej informacji na ten temat znajduje się w rozdziale System plików. Składnia: working_directory Zwroty: String Przykład: ini_open(working_directory + "temp_ini.ini"); Spowoduje to otwarcie pliku ini z katalogu roboczego gry (utworzenie go, jeśli jeszcze nie istnieje). Może to być lokalny magazyn lub obszar, w którym znajdują się dołączone pliki, w zależności od tego, czy szukany plik istnieje w którymś z tych miejsc.
