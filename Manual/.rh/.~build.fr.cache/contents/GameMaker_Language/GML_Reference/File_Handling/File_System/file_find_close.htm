file_find_close Funkcja ta musi być wywołana po obsłudze plików otwartych przy użyciu file_find_first() oraz file_find_next() aby zwolnić pamięć. Funkcje znajdowania plików otwierają uchwyty do katalogu plików i zajmują one minimalną ilość pamięci, która (z czasem) się sumuje. Dlatego należy zawsze wywoływać tę funkcję po znalezieniu potrzebnych plików, aby "zamknąć" te uchwyty. Składnia: file_find_close(); Zwraca Przykład: if directory_exists("\User Content") {     fileA = file_find_first("/User Content/*.doc", fa_readonly);     fileB = file_find_next();     fileC = file_find_next();     file_find_close(); } Ten kod sprawdza, czy podany katalog istnieje, a jeśli tak, to przechodzi do niego i zwraca pierwszy znaleziony plik "tylko do odczytu" .doc. Następnie szuka dwóch kolejnych plików i zamyka wyszukiwarkę plików. Można wyszukać dowolną liczbę plików za pomocą while pętli: var files = []; var file_name = file_find_first("/User Content/*.doc", fa_readonly); while (file_name != "") {     array_push(files, file_name);     file_name = file_find_next(); } file_find_close(); Powyższy kod tworzy pustą tablicę przechowującą wszystkie nazwy plików, które zostały znalezione, i rozpoczyna wyszukiwanie plików tylko do odczytu .doc. Jeśli dana nazwa pliku nie jest pustym string, przesuwa ją do tablicy files, a następnie próbuje znaleźć następny plik. Operacja loop będzie powtarzana do momentu znalezienia pustej tablicy string (co oznacza, że nie ma już pasujących plików), co spowoduje zakończenie operacji loop i zamknięcie wyszukiwarki plików na końcu.
