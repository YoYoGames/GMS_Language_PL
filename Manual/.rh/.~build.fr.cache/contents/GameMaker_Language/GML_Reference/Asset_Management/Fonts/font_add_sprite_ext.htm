czcionka_add_sprite_ext Za pomocą tej funkcji możesz użyć "paska sprite'ów" (sam sprite musi być sprite asset z przeglądarki Asset lub sprite dodanym do zasobów gry za pomocą sprite_add), aby utworzyć nowy font asset, w którym każdy podobraz będzie indywidualnym symbolem lub literą. W przeciwieństwie do zwykłego font_add_sprite() która ma określoną kolejność podobrazów w sprite, ta funkcja mapuje podobrazy sprite na podstawie argumentu "string_map" funkcji. Ten argument to adres string, którego można użyć, aby powiedzieć GameMaker Studio 2, w jakiej kolejności są podobrazy adresu sprite font, a funkcja będzie je automatycznie mapować podczas pisania tekstu. Jeśli więc na przykład masz mapę łańcuchów "AaBbCcDdEeFfGgHh", na stronie sprite font podobrazy muszą być uporządkowane w ten sposób. Możesz zdefiniować "spację" jako dowolny znak, na przykład pojedynczą linię o rozmiarze, jaki chcesz, aby miała spacja, a wszystkie spacje w tekście będą renderowane na tej szerokości (ale wybrany obraz nigdy nie będzie renderowany), lub jeśli nie podasz adresu sprite dla spacji, zamiast niej zostanie użyta szerokość najszerszego znaku w sprite font. Można także ustawić odstępy między literami font, aby były proporcjonalne (true) lub nie (false), przy czym odstępy proporcjonalne font są zależne od rzeczywistej szerokości liter (na przykład litera "i" zajmuje mniej miejsca room niż litera "w"), a odstępy nieproporcjonalne font są zależne od szerokości podobrazia, podobnie jak odstępy jednoprzestrzenne font. Na koniec można określić odstęp, jaki ma być pozostawiony między każdą literą podczas pisania. Może to być dowolna wartość całkowita, przy czym 0 oznacza brak odstępu (litery będą się "stykać", jeśli są proporcjonalne). Funkcja zwraca wartość indeksu, którą należy zapisać w zmiennej, ponieważ będzie ona potrzebna we wszystkich kolejnych kodach odwołujących się do tej strony font, lub zwraca -1, jeśli z jakiegoś powodu funkcja się nie powiedzie. Warto zauważyć, że aby działały funkcje wyrównywania font (np. draw_set_halign()), strona font sprite powinna mieć ustawiony początek w lewym górnym rogu. Jeśli użyjesz innych wartości, podczas rysowania tekstu za pomocą font będziesz musiał wziąć pod uwagę przesunięcie początku. UWAGA Po załadowaniu strony font do GameMaker Studio 2 należy pamiętać o jej ponownym usunięciu (za pomocą font_delete()), gdy nie jest już potrzebna, w przeciwnym razie istnieje ryzyko wycieku pamięci, który spowolni, a w końcu doprowadzi do awarii gry. Składnia: font_add_sprite_ext(spr, string_map, prop, sep); Argument Typ Opis spr Aktywa Sprite The sprite to add a font based on. string_map String String from which sprite sub-image order is taken. prop Boolean Set as proportional font or not. sep Real The space to leave between each letter. Zwroty: Font Asset Przykład: global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2); Powyższy kod utworzy nową stronę font asset na podstawie strony sprite zindeksowanej w zmiennej "spr_CalcFont" i zapisze indeks nowej strony font w zmiennej "global.Font".
