Stałe shadera Oprócz funkcji i stałych shader zdefiniowanych na stronach referencyjnych języka cieniowania OpenGL ES (GLSL ES), dostępnych jest także wiele stałych shader, które są unikalne dla GameMaker Studio 2. Następujące stałe matrycy wyświetlacza mogą być używane jako indeksy tablicy przy użyciu stałej tablicowej shader gm_Matrices : Stała Opis MATRIX_VIEW Ta stała indeksu tablicy przechowuje indeks bieżącej macierzy widoku. Zwrócony indeks zostanie użyty jako wartość tablicowa podczas wywoływania stałej gm_Matrices w kodzie shader. MATRIX_PROJECTION Ta stała indeksu tablicy przechowuje indeks bieżącej macierzy rzutowania. Zwrócony indeks zostanie użyty jako wartość tablicowa podczas wywoływania stałej gm_Matrices w kodzie shader. MATRIX_WORLD Ta stała indeksu tablicy przechowuje indeks do bieżącej macierzy świata. Może być używana do takich rzeczy jak oświetlenie, jeśli posiadasz informacje o świetle w przestrzeni świata. Zwrócony indeks zostanie użyty jako wartość tablicowa podczas wywoływania stałej gm_Matrices w kodzie shader. MATRIX_WORLD_VIEW Ta stała indeksu tablicy przechowuje indeks wyniku mnożenia macierzy świata i widoku. Jest to często używane do takich rzeczy, jak mgła. Zwrócony indeks zostanie użyty jako wartość tablicowa podczas wywoływania stałej gm_Matrices w kodzie shader. MATRIX_WORLD_VIEW_PROJECTION Ta stała indeksu tablicy przechowuje indeks wyniku mnożenia macierzy świata, widoku i rzutowania. Jest to normalna macierz transformacji używana dla pozycji vertex. Zwrócony indeks zostanie użyty jako wartość tablicowa podczas wywoływania stałej gm_Matrices w kodzie shader. MATRIX_MAX Nie jest to indeks tablicy, lecz raczej zwracany jest rozmiar tablicy macierzy w witrynie vertex shader. MAX_VS_LIGHTS Nie jest to indeks tablicy, ale raczej zwracana jest liczba świateł w witrynie vertex shader. Następująca stała jest również dostępna dla oświetlenia: Stała Opis MAX_VS_LIGHTS Maksymalna liczba świateł punktowych i kierunkowych dostępnych w shaderze W witrynie shader można używać następujących predefiniowanych uniformów i stałych macierzowych, aby uzyskać dostęp do wartości specyficznych dla witryny GameMaker Studio 2: Stała Opis gm_Matrices[matrix] Ta stała tablicowa zwraca macierz transformacji i jest jedną z dostępnych predefiniowanych uniformów, które GameMaker Studio 2 tworzy do użytku w edytorze kodu shader. Indeks tablicy jest wybierany z jednej z wyżej wymienionych stałych. gm_BaseTexture Jest to stała próbnika 2D, która zwraca wartość texture bieżącej strony object, ustawioną przez GameMaker Studio 2. Byłaby to więc bieżąca strona sprite, powierzchnia lub texture, która normalnie byłaby używana podczas rysowania bez wywoływania strony shader. gm_LightingEnabled Można go użyć do uzyskania lub ustawienia oświetlenia GameMaker Studio 2 podczas korzystania z funkcji 3D. gm_FogStart Można jej użyć do określenia odległości, przy której wielokąty zaczynają być mieszane z kolorem mgły. gm_RcpFogRange Można to wykorzystać do określenia odległości, przy której mgła jest największa i nic już nie widać. gm_PS_FogEnabled Zwróci ona wartość true lub false, jeśli układ GPU ma włączoną funkcję mgły pikselowej lub nie. gm_FogColour Można go użyć do uzyskania koloru mgły używanego przez stronę GameMaker Studio 2. gm_VS_FogEnabled Spowoduje to zwrócenie adresu true lub false, jeśli w układzie GPU jest włączona funkcja mgły vertex lub nie. gm_AlphaTestEnabled Można tego użyć, aby rozpocząć testy alfa na stronie shader. Zobacz gpu_set_alphatestenable() aby uzyskać więcej informacji na temat testów alfa. gm_AlphaRefValue Można go użyć do uzyskania bieżącej wartości referencyjnej testu alfa.
