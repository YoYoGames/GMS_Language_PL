Generowanie zdarzeń Object GameMaker Studio 2 jest w pełni sterowany zdarzeniami, co oznacza, że wszystkie działania są wynikiem pewnych zdarzeń, które są wywoływane w określonej kolejności lub przez określone zdarzenie w grze. Istnieje wiele różnych zdarzeń, z których każde ma określone zadanie lub czas trwania, a niektóre z nich są nawet podzielone na osobne zdarzenia "podrzędne". Zwykle zdarzenia te są uruchamiane przez stronę GameMaker Studio 2 po wykryciu zdarzenia wyzwalającego w grze lub w każdym ticku gry (tak jak w przypadku zdarzeń kroku i rysowania), ale można też zastosować zdarzenie do instancji w bieżącym obszarze room za pomocą fragmentu kodu. Istnieją następujące funkcje związane z generowaniem zdarzeń: event_inherited event_perform zdarzenie_perform_async event_perform_object event_user Istnieje również wiele zmiennych tylko do odczytu, które można wykorzystać z wyżej wymienionymi funkcjami, a które są szczególnie przydatne podczas debugowania: event_action event_number event_object event_type Więcej informacji na temat zdarzeń można znaleźć w rozdziale Zdarzenia na obiekcie.
