object_set_persistent Za pomocą tej funkcji można ustawić trwałość określonej strony object. Trwały object jest "przenoszony" z room na room i zachowuje swoje właściwości przez cały czas (więcej informacji o trwałości znajdziesz w rozdziale Edytor obiektów). Ustawienie trwałości dla object oznacza, że wszystkie instancje object, które zostaną utworzone po zmianie trwał ości, zostaną utworzone jako trwałe, podczas gdy instancje, które już znajdują się w room, mogą pozostać bez zmian. Gdy utworzysz instancję z włączoną opcją persystencji w witrynie room, zostanie jej przypisana warstwa lub głębokość (w zależności od tego, jakiej funkcji użyłeś do utworzenia instancji w witrynie object). Gdy room zostanie zmieniony i jeśli następna room nie będzie miała warstwy o tej samej nazwie lub głębokości, co warstwa przypisana, to dla instancji zostanie utworzona nowa warstwa, która będzie trwała przez rooms. Jeśli podczas tworzenia instancji nadałeś jej nazwę, to nowa warstwa również będzie miała taką samą nazwę jak warstwa oryginalna, a jeśli przypisałeś jej głębokość, to nowa warstwa będzie miała nazwę "_layer_XXX", gdzie "XXX" to wartość heksadecymalna służąca do nadania warstwie unikalnej nazwy. Warstwy te zostaną usunięte z room na Room End. Kolejną rzeczą, o której należy pamiętać w kontekście warstw i instancji trwałych, jest to, że jeśli podczas tworzenia instancji przypisałeś jej warstwę o tej samej nazwie, a w następnej części rooms znajduje się inna warstwa o tej samej nazwie, to instancja trwała zostanie przypisana do warstwy o tej samej nazwie, niezależnie od głębokości warstwy. Wreszcie, jeśli instancja uporczywa zostanie przeniesiona na stronę room z warstwą na tej samej głębokości, na której została utworzona, nie zostanie przypisana do tej warstwy, ale zamiast tego zostanie utworzona nowa warstwa na tej samej głębokości (zgodnie z konwencją nazewnictwa wyjaśnioną wcześniej). Należy pamiętać, że nie jest to funkcja instancji! Możesz ustawić persystencję flag poszczególnych instancji za pomocą zmiennej persistent i w ten sposób mieć dziesięć instancji oznaczonych jako trwałe, nawet jeśli object, o którym mowa, ma w flag ustawioną wartość IDE na false, i nawet jeśli zmienisz za pomocą tej funkcji wartość persystencji flag w object na true, wszystkie instancje znajdujące się obecnie w room pozostaną takie, jakie były, a tylko instancje utworzone po wywołaniu tej funkcji będą zaczynały z persystencją oznaczoną jako true. Składnia: object_set_persistent(index, pers); Argument Typ Opis index Obiekt Aktywa The index of the object to change. pers Boolean Whether the object should be flagged as persistent (true) lub nie (false). Zwroty: N/A Przykład: if (room == rm_final) {     object_set_persistent(obj_Player, false); } instance_create_layer(32, 32, "Instances", obj_Player); Powyższy kod sprawdza, w którym room znajduje się obecnie instancja i jeśli jest w określonym "rm_final", ustawia trwałość object "obj_Player" na false, a następnie tworzy instancję.
