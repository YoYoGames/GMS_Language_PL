asset_has_tags Za pomocą tej funkcji można sprawdzić, czy jeden lub więcej znaczników strings jest przypisanych do dowolnego asset z przeglądarki asset. Podaje się albo nazwę asset (jako string), albo jego indeks asset, a także pojedynczy znacznik string lub tablicę, w której każdy element jest pojedynczym znacznikiem string. Jeśli podasz wartość indeksu asset, musisz podać opcjonalny argument typu asset (stałą), ponieważ assets różnych typów mogą mieć ten sam indeks, choć nie mogą mieć tej samej nazwy. Dostępne typy asset są wymienione w poniższej tabeli: Stała Opis asset_object Podana nazwa odnosi się do obiektu. asset_sprite Podana nazwa odnosi się do sprite'a. asset_sound Nadana nazwa odnosi się do dźwięku. asset_room Nazwa odnosi się do room. asset_tiles Podana nazwa odnosi się do zestawu płytek. asset_path Podana nazwa odnosi się do ścieżki. asset_script Nazwa ta odnosi się do script. asset_font Podana nazwa odnosi się do czcionki. asset_timeline Imię odnosi się do linii czasu. asset_shader Podana nazwa odnosi się do shadera. asset_animationcurve Podana nazwa odnosi się do krzywej animacji. asset_sequence Podana nazwa odnosi się do sekwencji. Jeśli funkcja powiedzie się i wszystkie podane znaczniki są obecne dla asset, zwróci true, w przeciwnym razie zwróci false. Jeśli chcesz sprawdzić obecność dowolnego z wybranych znaczników, a nie wszystkich, użyj funkcji asset_has_any_tag(). Składnia: asset_has_tags(name_or_index, tags, [asset_type]); Argument Typ Opis name_or_index Ciąg lub aktywa The name of the asset (a string) or its index value (an integer). tags Łańcuch lub tablica łańcuchów A single asset tag string or an array with various asset tags. [asset_type] Rodzaj aktywów Stała OPTIONAL! The type of asset to check the tags for, only used when supplying an index for the first argument. Zwroty: Boolean Przykład: var _a = array_create(3); _a[0] = "enemy"; _a[1] = "level_" + string(global.Level); _a[2] = "boss"; if asset_has_tags(object_index, _a, asset_object) {     instance_create_layer(0, 0, "Overlay", obj_Boss_Text); } Powyższy kod utworzy tablicę znaczników, a następnie sprawdzi, czy wszystkie zostały zastosowane do podanej strony object, a jeśli tak, to utworzy instancję innej strony object.
