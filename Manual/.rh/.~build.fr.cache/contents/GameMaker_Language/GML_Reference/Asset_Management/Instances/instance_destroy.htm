zniszczyć instancję Wywołujesz tę funkcję za każdym razem, gdy chcesz "zniszczyć" instancję, zwykle wywołując zdarzenie Destroy i zdarzenie Clean Up. Spowoduje to usunięcie jej z room do czasu ponownego uruchomienia room (chyba że room jest trwałe). Wywołanie funkcji bez argumentów po prostu zniszczy instancję, która jest aktualnie w zasięgu i uruchamia kod, ale możesz podać opcjonalny argument "id" i skierować się do konkretnej instancji, używając wartości instance id, lub możesz skierować się do wszystkich instancji danego object, używając wartości . object_index. Na przykład: instance_destroy(other);      // destroy the "other" instance in a Collision Event instance_destroy(obj_Bullet); // destroy ALL instances of the object "obj_Bullet" Drugi opcjonalny flag pozwala "wyłączyć" zdarzenie Destroy dla niszczonej instancji. Domyślnie zdarzenie Destroy będzie zawsze wykonywane, ale jeśli ustawisz tę opcję flag na false, możesz sprawić, że instancja sama się zniszczy i pominąć wykonywanie tego zdarzenia. WAŻNE! Spowoduje to pominięcie zdarzenia Zniszcz, ale zdarzenie Wyczyść nadal będzie wywoływane. Warto zauważyć, że gdy niszczysz instancję, jej zdarzenie destroy jest wywoływane natychmiast po kodzie lub akcji, która wywołuje funkcję destroy. Ponadto, mimo że zdarzenie destroy jest wykonywane, instancja nie jest natychmiast usuwana z gry i będzie nadal wykonywać kod zawarty w bieżącym zdarzeniu. Dopiero po zakończeniu bieżącego zdarzenia instancja zostanie usunięta z gry. Jeśli więc masz na przykład taki kod: if (hp <= 0) instance_destroy(); score += 10; Zmienna "score" będzie zwiększana, mimo że została wywołana funkcja instance_destroy , a instancja zostanie ostatecznie usunięta z gry na końcu zdarzenia. Uważaj na to, ponieważ jeśli (na przykład) utworzyłeś dynamiczny zasób dla instancji, taki jak struktura danych, a następnie zniszczyłeś ten zasób w zdarzeniu destroy, to jeśli po wykonaniu funkcji lub akcji destroy istnieją jakiekolwiek odniesienia do tego zasobu, otrzymasz błąd "nieznany zasób", ponieważ zdarzenie destroy usunęło go z gry. Składnia: instance_destroy([id, execute_event_flag]); Argument Typ Opis id Identyfikator instancji lub Obiekt Asset OPTIONAL ID instancji do zniszczenia lub object asset do zniszczenia instancji (opcjonalne, domyślnie instancja wywołująca) execute_event_flag Boolean OPTIONAL Ustaw na true lub false, aby wykonać zdarzenie Destroy lub nie (opcjonalne, domyślnie true). Zwroty: N/A Przykład: if (bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height) {     instance_destroy(id, false); } Powyższy kod zniszczy instancję wykonującą kod bez wywoływania zdarzenia Destroy, jeśli instancja znajduje się poza granicami room.
