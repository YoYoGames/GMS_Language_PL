audio_debug Tej funkcji można używać do wyświetlania informacji o systemie audio, włączając ją za pomocą adresu true, a wyłączając za pomocą adresu false lub switch. W przypadku korzystania w ten sposób z funkcji debugging audio na nakładce będą wyświetlane następujące informacje: Handle - uchwyt źródła OpenAL (unikalny dla każdego odtwarzanego dźwięku) Data Handle - uchwyt OpenAL buffer (pokazuje tę samą wartość dla wielu dźwięków z tym samym uchwytem źródłowym) Przetworzone bufory - ile stron buffers zostało odtworzonych Queued Buffers - ile stron buffers zostało ustawionych w kolejce (i nadal czeka na odtworzenie) Wzmocnienie - wartość wzmocnienia dla dźwięku (0-65536) Play Offset - pozycja próbki (bieżąca pozycja odtwarzania) w bieżącym buforze Paski kolejki Buffers - mały diagram pudełkowy przetwarzanej/ oczekującej strony buffers z przesunięciem odtwarzania (bieżąca pozycja) narysowanym jako niebieska linia W przypadku zsynchronizowanych grup dźwięków dostępna jest również funkcja audio_sync_group_debug(). UWAGA Ta funkcja nie działa na platformie docelowej HTML5. Składnia: audio_debug(enable); ArgumentTyp Opis enable Enable (set to true) lub wyłączenie (false) dźwięku debugging. Zwroty: Przykład: if (debug_mode) {     audio_debug(true); } else {     audio_debug(false); } Powyższy kod na stronie switch włączy lub wyłączy nakładkę debugowania dźwięku w zależności od tego, czy gra działa w trybie debugowania, czy nie.
