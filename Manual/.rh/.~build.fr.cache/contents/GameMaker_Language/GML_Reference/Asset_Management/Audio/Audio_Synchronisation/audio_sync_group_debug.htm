audio_sync_group_debug Za pomocą tej funkcji można wyświetlić informacje debugowania dotyczące dowolnej grupy synchronizacji (indeks grupy jest wartością zwracaną podczas tworzenia grupy za pomocą funkcji audio_create_sync_group()). Funkcję można wywołać z wartością -1, aby switch wyłączyć nakładkę. Podczas korzystania z witryny debugging grupy synchronizacji dźwięku są wyświetlane na nakładce: Która grupa synchronizacji jest debugowana Ile jest w nim kanałów Ile razy grupa synchronizacji zakończyła się niepowodzeniem (oznacza to, że próbowała wykonać zbyt wiele zadań) Lista kanałów Czerwony oznacza, że ten kanał nie wykonuje obecnie żadnej pracy Żółty oznacza, że kanał jest w stanie początkowym (jeszcze nie dekoduje ani nie odtwarza) Zielony oznacza, że ścieżka jest w trakcie wyszukiwania/dekodowania Fioletowy oznacza, że w kanale skończyły się dane. Pasek profilowania, który pokazuje, jaką pracę wykonuje wątek dekodujący kolor niebieski oznacza, że plik jest odczytywany po raz pierwszy czerwony oznacza, że wątek czyta plik, aby dowiedzieć się, skąd ma odczytać dane zielony oznacza, że wątek dekoduje dane fioletowy oznacza, że wątek jest uśpiony do czasu, gdy będzie musiał pracować Należy pamiętać, że dla ogólnej strony debugging audio dostępna jest także funkcja audio_debug(). UWAGA: Ta funkcja nie jest dostępna dla platformy docelowej HTML5. Składnia: audio_sync_group_debug(group_index); Argument Typ Opis group_index ID grupy synchronizacji audio The group index to debug, or -1 to switch off the debugging. Zwroty: N/A Przykład: if (debug_mode) {     audio_sync_group_debug(sg); } else {     audio_sync_group_debug(-1); } Powyższy kod na stronie switch włączy lub wyłączy nakładkę debugowania grupy synchronizacji dźwięku, pokazującą grupę synchronizacji zindeksowaną w zmiennej "sg", w zależności od tego, czy gra jest uruchomiona w trybie debugowania, czy nie.
