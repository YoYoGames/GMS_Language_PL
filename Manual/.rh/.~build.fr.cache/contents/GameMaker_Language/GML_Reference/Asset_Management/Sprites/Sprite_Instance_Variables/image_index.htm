image_index Strona sprite składa się z jednego lub kilku pod-obrazów, które mogą sprawić, że strona sprite będzie animowana, gdy switch będą przechodzić z jednego do drugiego, lub można je przełączać w kodzie, aby uzyskać różne "stany", podobnie jak przycisk w oknach. Jeśli sprite jest animowany, możesz uzyskać bieżącą klatkę animacji, sprawdzając zmienną image_index, lub jeśli chcesz zmienić stan statycznego sprite, możesz wybrać nowy podobraz, ustawiając tę zmienną na żądany podobraz sprite. Należy pamiętać, że aby zmiany w tej zmiennej były widoczne, instancja nie powinna mieć zdarzenia draw (a więc GameMaker Studio 2 domyślnie narysuje sprite) lub być rysowana za pomocą jednej z funkcji rysowania, takich jak draw_self() lub draw_sprite_ext() (przez podanie image_index w odpowiednim argumencie). Należy pamiętać, że ustawienie tej zmiennej bezpośrednio na ramkę nie spowoduje wyzwolenia żadnych komunikatów nadawczych, które mogą być w niej obecne. Należy także pamiętać, że podczas używania animacji szkieletowej sprites można nadal uzyskiwać i ustawiać wartości image_index - zobacz funkcję skeleton_animation_get_duration() zawiera przykłady, jak to robić. Składnia: image_index; Zwroty: (single precision floating point value) Przykład: if (image_speed > 0) {     if (image_index >= image_number - 1) instance_destroy(); } Powyższy kod sprawdza, czy strona sprite jest animowana, a jeśli tak, to sprawdza, czy bieżący image_index znajduje się w ostatniej klatce i w takim przypadku niszczy instancję.
