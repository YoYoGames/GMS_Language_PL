sprite_add_from_surface Ta funkcja działa dokładnie tak samo jak sprite_create_from_surface() tylko zamiast tworzyć nowy obraz sprite z wybranej powierzchni indeksowanej, dodaje zdefiniowany obszar powierzchni jako nowy podobraz do wcześniej utworzonego obrazu sprite (oznacza to, że nie można go dodać bezpośrednio do obrazu sprite z drzewa zasobów, lecz tylko do obrazu utworzonego wcześniej z powierzchni lub powielonego z zasobu sprite za pomocą sprite_duplicate()). Należy zauważyć, że pamięć używana podczas dodawania strony sprite w ten sposób będzie większa, niż można się spodziewać. Dzieje się tak, ponieważ witryna GameMaker Studio 2 zapisze stronę sprite jako stronę texture i będzie ona również przechowywana w pamięci GPU, więc całkowita ilość pamięci będzie większa niż w przypadku pliku graficznego o tej samej nazwie sprite. Składnia: sprite_add_from_surface(index, surface, x, y, w, h, removeback, smooth); Argument Typ Opis index Aktywa Sprite The index of the sprite to add the new image to. surface Identyfikacja powierzchni The index of the surface from which we get the image. x Real The x position to copy from. y Real The y position to copy from. w Real The width of the area to be copied (from the x position). h Real The height of the area to be copied (from the y position). removeback Boolean Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent. smooth Boolean Indicates whether to smooth the edges. Zwroty: N/A Przykład: spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16); var i; for (i = 1; i < 8; i += 1) {     sprite_add_from_surface(spr_custom, surf, i, 0, 32, 32, true, true); } Powyższy kod tworzy stronę sprite z powierzchni indeksowanej w zmiennej "surf", przypisując jej indeks do zmiennej "spr_Custom", a następnie używa strony for loop do poruszania się po powierzchni i przechwytywania różnych fragmentów, które są dodawane do strony sprite jako obrazy podrzędne.
