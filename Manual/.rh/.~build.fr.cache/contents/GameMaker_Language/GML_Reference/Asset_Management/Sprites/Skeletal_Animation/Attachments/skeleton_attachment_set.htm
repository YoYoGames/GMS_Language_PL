szkielet_załącznika_set Do animacji szkieletowej sprite można dodawać inne obrazy jako załączniki, przy czym obrazy te są dodawane do nazwanego slotu (nazwa jest podawana podczas tworzenia slotu załącznika w programie do animacji) i będą rysowane wraz z animacją bieżącego sprite. Za pomocą tej funkcji można ustawić załącznik do danego slotu, przy czym wymagane jest podanie nazw (jak strings) slotu i załącznika. Nazwy te są definiowane przez używany program animacji lub (w przypadku przystawki) podczas wywołania funkcji skeleton_attachment_create(). Można również podać sprite_index jako załącznik, wtedy użyty zostanie sprite, lub użyć -1, aby usunąć załącznik ze slotu. Jeśli jako argument podamy indeks sprite, zostanie utworzony slot załącznika, używając nazwy sprite jako string do nazwania slotu (więc na przykład użycie spr_sword utworzy slot załącznika "spr_sword"), a slot użyje pierwszego indeksu obrazu (0) z sprite, jak również jego x i y origin offset, ze skalą (1,1) i rotacją 0. Składnia: skeleton_attachment_set(slot, attachment); Argument Typ Opis slot String The slot name (a string) to get the attachment of. attachment String lub Sprite Asset The name (as a string or a sprite_index) of the attachment image. Zwroty: String Przykład: if skeleton_attachment_get("slot_leftHand") == "" {     skeleton_attachment_set("slot_leftHand", choose("sword", "spear", "knife")); } Powyższy kod sprawdzi aktualnie przypisaną nazwę załącznika dla slotu o nazwie "slot_leftHand" i jeśli zwrócona zostanie pusta string, zostanie dołączony nowy sprite.
