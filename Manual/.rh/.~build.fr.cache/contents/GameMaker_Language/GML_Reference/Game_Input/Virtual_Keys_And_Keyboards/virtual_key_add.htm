virtual_key_add Ta funkcja umożliwia mapowanie "dotknięć" obszaru ekranu na zdarzenia na klawiaturze. Oznacza to, że po przypisaniu obszaru do klawisza wirtualnego wszystkie dotknięcia tego obszaru będą wywoływać zdarzenie klawiaturowe odpowiadające klawiszowi, który został zmapowany do tego obszaru. Każdy zdefiniowany klawisz wirtualny można przypisać do zmiennej, która może być następnie użyta w dalszych funkcjach klawiszy wirtualnych do ich pokazywania, ukrywania i usuwania. Klawisze te są przypisywane do poszczególnych pomieszczeń i będą automatycznie usuwane przez stronę GameMaker Studio 2 przy zmianie adresu rooms. Rzeczywiste położenie klucza wirtualnego jest oparte na pozycji ekranu, a nie na pozycji room, dlatego wartości x/y są bezwzględne na ekranie. Oznacza to, że nie trzeba się martwić o używanie widoków ani względnych współrzędnych room. Można po prostu zdefiniować klucze w zdarzeniu Utwórz w witrynie object (wystarczy zdefiniować klucz wirtualny tylko raz dla danego pomieszczenia), a następnie narysować klucz sprites w zdarzeniu Narysuj GUI, ustawiając warstwę GUI tak, aby była taka sama jak ekran. Składnia virtual_key_add(x, y, w, h, keycode); Argument Typ Opis x Real The x coordinate (left side) of the virtual key on the screen y Real The y coordinate (top side) of the virtual key on the screen w Real The width of the virtual key h Real The height of the virtual key keycode Stała klucza wirtualnego (vk_*) Which keyboard key event should be triggered by touching this area Zwroty: Virtual Key ID Przykład: global.Left = virtual_key_add(32, 32, 64, 64, vk_left); Powyższy kod tworzy wirtualny klucz o wymiarach 64x64 pikseli, umieszczony na ekranie w punkcie (32, 32), który będzie wyzwalał zdarzenie vk_left po dotknięciu i przypisuje indeks tego wirtualnego klucza do zmiennej globalnej.
