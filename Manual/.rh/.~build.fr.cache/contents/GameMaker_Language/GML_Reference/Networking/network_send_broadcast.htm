network_send_broadcast Za pomocą tej funkcji można rozgłaszać dane z adresu buffer lokalnie do zakresu adresów IP (więcej informacji na temat adresu buffers można znaleźć w części Odnośnik - Bufory). Zakres jest ograniczony do zakresu urządzenia, na którym działa server, np. jeśli urządzenie ma adres IP 92.168.11.130, to dane zostaną rozesłane na zakres 92.168.11.*. Funkcja zwraca liczbę bajtów wysłanych danych lub liczbę mniejszą od 0, jeśli wysyłanie nie powiodło się. UWAGA: Ta funkcja będzie działać tylko w przypadku korzystania z protokołu UDP - witryna server musi być podłączona do protokołu TCP, a klient musi mieć gniazdo klienta UDP utworzone za pomocą protokołu network_create_socket_ext() aby móc odbierać wszelkie transmisje wysyłane z server. UWAGA: Ta funkcja nie będzie działać, jeśli zostanie użyta w projekcie działającym na platformie docelowej HTML5. Składnia: network_send_broadcast(socket, port, bufferid, size); ArgumentTyp Opis socket The id of the socket to use. port The port that the server will use. bufferid The id of the buffer to get the data from. size The size (in bytes) of the data. Zwroty: Real Przykład: buffer_seek(broadcast_buffer, buffer_seek_start, 0); buffer_write(broadcast_buffer, buffer_string, global.ServerName); network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer)); Powyższy kod zapisuje nazwę string bieżącego adresu server (przechowywaną w pliku "global.ServerName"), a następnie zapisuje ją w pliku binarnym buffer o identyfikatorze "broadcast_buffer". Dane te są następnie rozgłaszane lokalnie do szeregu adresów IP (adres IP urządzenia jest obecnie implikowany jako podstawowy zakres rozgłaszania) na porcie 6511.
