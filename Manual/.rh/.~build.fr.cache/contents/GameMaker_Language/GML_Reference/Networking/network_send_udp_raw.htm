network_send_udp_raw Za pomocą tej funkcji można wysyłać dane przez sieć przy użyciu protokołu UDP na adres server. Funkcja pobiera identyfikator gniazda, przez które ma się łączyć, adres URL, z którym ma się łączyć, oraz port, którego ma używać. Następnie należy podać identyfikator bufora, który zawiera dane do wysłania (więcej informacji na ten temat można znaleźć na stronie buffers w rozdziale Reference - Buffers) oraz rozmiar (w bajtach) danych. UDP jest "bezpołączeniowy" w tym sensie, że w rzeczywistości nie wykonuje się połączenia, tylko wysyła pakiet bezpośrednio do adresu IP, a server otrzymuje dane przychodzące z adresu IP i musi sobie z nimi poradzić "tak jak jest". Wysyłane dane nie są w żaden sposób formatowane przez GameMakera, a urządzenia odbierające otrzymają je w postaci strumienia, co oznacza, że będziesz musiał sobie z nimi poradzić. Funkcja zwraca liczbę bajtów wysłanych danych lub liczbę mniejszą od 0, jeśli wysyłanie nie powiodło się. UWAGA Ta funkcja nie będzie działać, gdy będzie używana w projekcie działającym na platformie docelowej HTML5, a projekty HTML5 nie będą mogły odbierać UDP. Składnia: network_send_udp_raw(socket, url, port, bufferid, size); ArgumentTyp Opis socket The id of the socket to use. url The url or IP to connect to (a string). port The port to connect to. bufferid The id of the buffer to get the data from. size The size (in bytes) of the data. Zwroty: Real Przykład: network_send_udp_raw(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff)); Powyższy kod wyśle nieprzetworzony pakiet UDP na port 6510 na adres server określony przez adres URL.
