Pliki wsadowe / skrypty kompilatora Kompilator GameMakera pozwala na umieszczenie plików Batch (Windows) lub Shell scripts (na macOS i Linux) w katalogu projektu oraz w katalogu dla każdego extension w projekcie. Placówka Plik script można umieścić w głównym katalogu projektu, obok pliku .yyp projektu. Plik script można również umieścić w katalogu extension, który może wyglądać następująco: /extensions//. Wersje extension pliku script będą wykonywane jako pierwsze, a jego wersja główna jako ostatnia. Nie można kontrolować kolejności wykonywania extensions, dlatego programista extension musi zadbać o to, aby scripts jednego rozszerzenia nie opierało się na scripts innego rozszerzenia. Wykonanie Pliki skryptów o określonych nazwach, umieszczone w dowolnym z opisanych powyżej katalogów, są wykonywane w różnych punktach procesu kompilacji: Nazwa pliku wsadowego systemu Windows powłoka systemu macOS/Linux Script Nazwa Opis pre_run_step.bat pre_run_step.sh Polecenie to jest wykonywane przed rozpoczęciem wdrażania gry post_run_step.bat post_run_step.sh Jest on wykonywany, gdy gra jest przygotowana i gotowa do uruchomienia; po wykonaniu skryptu gra jest uruchamiana pre_build_step.bat pre_build_step.sh Polecenie to jest wykonywane przed uruchomieniem kompilatora asset w celu zbudowania gry post_build_step.bat post_build_step.sh Polecenie to jest wykonywane po uruchomieniu kompilatora asset w celu zbudowania gry pre_package_step.bat pre_package_step.sh Czynność ta jest wykonywana przed ostatecznym etapem pakowania, czyli wtedy, gdy wszystkie pliki są gotowe, ale ma zostać utworzony ostateczny plik ZIP lub paczka sklepowa. post_package_step.bat post_package_step.sh Czynność ta jest wykonywana po zakończeniu ostatniego etapu pakowania i przygotowaniu ostatecznego pliku ZIP lub pakietu sklepowego. post_textures.bat post_textures.sh Czynność ta jest wykonywana po wygenerowaniu strony textures dla Twojej gry. Możesz użyć tego kroku do ręcznej optymalizacji gry textures. Ta strona script otrzymuje zmienną środowiskową o nazwie TexturesDir, która przechowuje adres path do wygenerowanej strony textures. Te pliki script są obsługiwane podczas budowania na wszystkich platformach docelowych. Wyłączanie strony Scripts dla rozszerzenia Jeśli używasz wielu stron extensions, na których działa scripts, na przykład Steamworks i GDK, to podczas eksportowania do GDK należy wyłączyć Steamworks i na odwrót.
