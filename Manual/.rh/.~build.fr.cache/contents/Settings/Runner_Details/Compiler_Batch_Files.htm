Pliki wsadowe / skrypty kompilatora Kompilator GameMaker Studio 2 pozwala na umieszczenie plików Batch (Windows) lub Shell scripts (na macOS i Linux) w katalogu projektu i wykonuje je w różnych punktach procesu kompilacji, jak opisano poniżej: Nazwa pliku wsadowego systemu Windows powłoka systemu macOS/Linux Script Nazwa Opis pre_run_step.bat pre_run_step.sh Polecenie to jest wykonywane przed rozpoczęciem wdrażania gry post_run_step.bat post_run_step.sh Jest on wykonywany, gdy gra jest przygotowana i gotowa do uruchomienia; po wykonaniu skryptu gra jest uruchamiana pre_build_step.bat pre_build_step.sh Polecenie to jest wykonywane przed uruchomieniem kompilatora asset w celu zbudowania gry post_build_step.bat post_build_step.sh Polecenie to jest wykonywane po uruchomieniu kompilatora asset w celu zbudowania gry pre_package_step.bat pre_package_step.sh Czynność ta jest wykonywana przed ostatecznym etapem pakowania, czyli wtedy, gdy wszystkie pliki są gotowe, ale ma zostać utworzony ostateczny plik ZIP lub paczka sklepowa. post_package_step.bat post_package_step.sh Czynność ta jest wykonywana po zakończeniu ostatniego etapu pakowania i przygotowaniu ostatecznego pliku ZIP lub pakietu sklepowego. Te pliki wsadowe/powłoka scripts są obsługiwane na wszystkich platformach docelowych.
